## 🏗️ Infrastructure

### Hardware
- **الكمبيوتر الرئيسي:** جهازك المحلي (Windows) — Docker + Next.js
- **Powerhouse Nodes:** Docker containers (headless Linux) بـ Playwright
- **Mobile:** Xiaomi Second Space (واجهة منفصلة) — Scrcpy wireless
- **Internet:** نفس شبكة WiFi (كمبيوتر + موبايل)

### Free Stack ($0.00)
| المكون | البديل المجاني | المدفوع اللي اتشال |
|--------|---------------|-------------------|
| AI/LLM | Ollama (local) | GPT-4, Claude, Gemini APIs |
| Image Generation | Stable Diffusion (local) | DALL-E, Midjourney |
| Voice TTS/STT | Web Speech API (browser) | Google TTS/STT |
| OCR | Tesseract.js (local) | Google Vision API |
| Email | Local SMTP / MailHog | Resend, Postmark |
| WhatsApp | whatsapp-web.js | WhatsApp Business API |
| Storage | Local filesystem | Cloud Blob Storage |
| Notifications | Browser Push API | Firebase Cloud Messaging |
| E-Signatures | Self-signed PDFs | DocuSign |
| LLM Models | llama3.2, mistral, codellama | GPT-4 |

### Security
- Local encryption (AES-256)
- Self-hosted = you own your data
- No third-party access
- Docker isolation (read-only root, memory limits 2GB, CPU 1.5, daily rebuild)
- Approval gates for social actions
- Resource locking (prevent conflicts)

## 🗄️ Database Schema (40+ جدول)

### Agent Tables
- `agent_profiles` — PRIME + Vanguard definitions
- `agent_devices` — Powerhouse-Alpha, Powerhouse-Beta, Xiaomi-Space
- `agent_device_heartbeats` — Real-time status (CPU, RAM, tasks)

- `agent_task_runs` — Execution history
- `agent_conversations` — Chat sessions (direct, group, agent-to-agent)
- `agent_messages` — Messages with @mentions
- `agent_handoffs` — Context transfer between agents
- `agent_events` — Real-time feed (stuck, completed, error)
- `agent_learnings` — Self-evolution knowledge
- `agent_memory` — Long-term storage

### Manufacturing Tables
- `sales_orders` — Post-contract manufacturing orders
- `sales_order_items` — Individual items in an order
- `payment_schedules` — Installment plans
- `production_stages` — Workflow stages (Measurement → Design → Cutting → Assembly → Finishing → QA → Packaging → Delivery)
- `production_jobs` — Individual manufacturing jobs
- `production_job_events` — History log
- `production_schedule_entries` — Gantt chart data
- `design_versions` — Design iterations with 3D data
- `bom_headers` — Bill of Materials root
- `bom_items` — Individual material items
- `inventory_items` — Master stock list
- `inventory_movements` — Stock ledger
- `inventory_reservations` — Materials reserved for jobs
- `purchase_orders` — Supplier orders
- `purchase_order_items`
- `quality_checks` — Photo upload + AI assessment
- `delivery_records` — Transport + installation

### Owner Control Tables
- `owner_policies` — Approval thresholds, budget limits
- `emergency_stop_state` — Kill switch state
- `alert_rules` — Custom notification rules
- `daily_digests` — Morning reports

### Core Business Tables (Existing)
- `companies` — Canonical root
- `users` — Actors (with auth_user_id FK)
- `team_roles` + `team_members` — RBAC

- `site_settings` — Configuration
- `audit_log` — Canonical audit ledger

```

  → Production Schedule → Material Order → Manufacturing
  → Quality Check → Delivery → Installation → Final Payment
  → Review → After-Sales Service

PRIME manages: Measure (if digital) → Design → Quote → Manufacturing → Quality
```

1. PRIME finishes design → notifies Vanguard
2. Vanguard presents to customer → gets approval
3. PRIME executes production
4. Vanguard schedules delivery
5. PRIME does quality check
6. Vanguard handles final payment

## ⚙️ Setup Instructions

### Prerequisites
1. Docker Desktop installed
2. Node.js 20+ installed
3. Ollama installed (for Week 7)
4. Stable Diffusion installed (for Week 7)
5. Supabase project created
6. Xiaomi Second Space enabled
7. Scrcpy installed: `choco install scrcpy`

### Environment Variables (.env.local)
```
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
ENCRYPTION_PASSWORD=your_strong_password_32_chars
LOCAL_LLM_URL=http://localhost:11434
STABLE_DIFFUSION_URL=http://localhost:7860
```

### Package.json Scripts
```json
{
  "scripts": {
    "supabase:push": "supabase db push",
    "supabase:reset": "supabase db reset",
    "db:types": "supabase gen types typescript --project-id $PROJECT_ID > lib/dal/database.types.ts",
    "db:migrate": "npm run supabase:push && npm run db:types",
    "build:check": "tsc --noEmit && next build"
  }
}
```

### Installation Steps
```bash
# Week 0: Pre-implementation
npm run db:migrate  # Run reconciliation + new tables

# Week 1-10
npm install
npm run build:check
docker-compose up -d
npm run dev
```
